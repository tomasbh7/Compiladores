#Branch strategy

I will create a branch for each practical assignment and for the final project with the final version of the practical assignment/project. Each practical assignment/project branch will have its own sub-branch in case it serves as test code or in case I want to test some things or make changes to the code that could affect the final result on a smaller or larger scale.
